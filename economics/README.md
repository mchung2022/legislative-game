# 國中社會/公民課教育遊戲——「市場大師：供需平衡大挑戰！」

這是一款專為**國中生**設計的網頁互動素養思辨遊戲，將經濟學中最核心的「供給與需求」與「公平交易法」概念包裝成趣味十足的創業與市場管理關卡。配合 108 課綱公民與社會科經濟單元，學生透過扮演市場經營與決策者、解答情境問答、微調供需拉桿、並進行公平競爭規章修正，加深對市場機制運作與政府干預後果的學習記憶。

---

## 🎮 遊戲特色與關卡流程

遊戲共有五大核心關卡，完整映射市場運作與供需機制：

1. **起步與供需收集（開市交易媒合小遊戲）**
   - 學生需在不同定位（高價奢華、國民折衷、低價薄利多銷）的經營方針中做出抉擇，影響初始的消費者、生產者與社會滿意度。
   - 隨後在限時內快速點擊飄浮的「買方/賣方交易氣泡」，媒合至少 **15 筆交易訂單**以順利開啟商品市場。
   - 點擊發光的議事槌，鳴鑼宣告正式開市！
2. **供需調和與抉擇（公聽會代表回應 + 供需平衡拉桿）**
   - 面對消費者（要求降價與退換權）、生產者（原料成本上漲與利潤）與政府官員（食安與資安等社會外部成本）的衝突訴求，做出關鍵決策。
   - 隨後進入價格協商，調整價格、偏好與成本滑桿，使「需求量（買方）」與「供給量（賣方）」兩大指針重疊在發光的「市場均衡區」，達成市場均衡。
3. **供需法則大考驗（經濟學素養答辯 + 電子交易媒合）**
   - 挑戰 10 題國中經濟學素養選擇題，內容涵蓋：需求法則、供給法則、市場均衡、短缺與過剩、替代品與互補品、非價格因素（偏好與成本變動）、最高限價與最低限價等。
   - 每答對一題能自動撮合 8 位成功交易者（亮起綠燈），答對率與民意支持度越高，電子交易媒合越容易成功（達 **57 人**以上成交門檻即可過關）。
4. **市場公平交易法（规章合理性校對與修正）**
   - 閱讀擬定的市場規章草案，找出草案中「聯合行為限制定價」或「濫用行政權強制拘留」的違法/違憲條款。
   - 點擊紅色下底線字，在彈出視窗中選擇正確的「公平競爭修正條款」，以維護自由市場交易秩序。
5. **供需危機與挑戰（突發衝擊危機處理 + 快速點擊穩定物價）**
   - 市場突發危機（手搖飲食安風暴、電競手機晶片大火、有機生菜颱風天災），面臨供需失衡價格狂飆！
   - 玩家在折衷、管制與放棄等危機處理策略中抉擇。若選擇管制或折衷，需在 8 秒內快速狂點「穩定市場」按鈕累積指數，高於 57 點即可成功度過衝擊！
6. **頒發證書與雲端同步**
   - 度過危機後，遊戲會自動結算學生的「供需平衡度」、「供需法則思辨」與「公平交易修正」指標。
   - 產生精美的「經濟素養市場大師證書」，供學生下載或列印提交給老師。
   - 同時，學生的姓名、班級、座號、作答成績與決策，將會自動非同步 POST 傳送到教師的 Google 試算表中！

## 🛠️ 技術架構

本專案本著**輕量、流暢、免安裝、100% 離線可用**的原則開發：
- **HTML5 & CSS3**：建立響應式玻璃擬物化 (Glassmorphism) 介面，支援手機與平板（方便課堂人手一台平板操作）。
- **Vanilla ES6+ JS**：無外部框架依賴，極速載入。
- **Web Audio API**：利用瀏覽器音頻引擎**即時合成遊戲音效**（包含按鍵聲、泡泡聲、木槌敲擊聲、答對升調、答錯降調、勝利歡呼噪聲），不需要加載任何外部音訊檔案，徹底避免因路徑錯誤或跨域阻擋導致音效失效。
- **Canvas**：用於結算頒發證書時的五彩紙屑 (Confetti) 特效模擬。

## 🚀 本地執行方式

本專案無需任何建置工具，雙擊即可執行：
1. 下載專案資料夾後，找到 `index.html`。
2. 在您的瀏覽器（建議使用 Chrome、Edge 或 Safari）中**按兩下開啟 `index.html`**，即可開始遊玩！

## 🌐 部署至 GitHub Pages

為了讓學生與老師能直接用手機或平板連線上網遊玩，您可以將此專案部署到 GitHub Pages（完全免費）：

### 步驟 1：建立 GitHub 倉庫 (Repository)
1. 登入您的 [GitHub](https://github.com/) 帳號。
2. 點擊右上角 **New** 新增一個儲存庫。
3. 儲存庫名稱（Repository name）可以輸入例如：`supply-demand-game`。
4. 選擇 **Public**（公開），然後點擊 **Create repository**。

### 步驟 2：上傳專案程式碼
在您本地的專案資料夾路徑下執行以下 Git 指令：
```bash
git init
git add .
git commit -m "feat: design Economics Supply and Demand game for junior high students"
git branch -M main
git remote add origin https://github.com/您的帳號名稱/supply-demand-game.git
git push -u origin main
```
*如果不熟悉指令，也可以直接在 GitHub 網頁上選擇「uploading an existing file」，將所有檔案拖曳上傳並 Commit。*

### 步驟 3：啟用 GitHub Pages
1. 在您的 GitHub 儲存庫頁面，點選上方的 **Settings**（設定）。
2. 在左側選單中找到並點選 **Pages**。
3. 在 **Build and deployment** 下的 **Source** 選擇 `Deploy from a branch`。
4. 在 **Branch** 下拉選單中選擇 `main` 分支與 `/ (root)` 資料夾，並點選右側的 **Save**（儲存）。
5. 等待約 1 到 2 分鐘，重新整理頁面，您就會在上方看到您的專案專屬網址（例如：`https://您的帳號名稱.github.io/supply-demand-game/`）。複製該網址，就可以分享給學生們使用了！
